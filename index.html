1 Introduccion

1.1 introduccion

Este documento es una Especificación de Requisitos de Software (ERS) para el sistema de información de una tienda virtual de productos tecnológicos. Esta especificación se ha estructurado basándose en las directrices dadas por el estandar IEEE práctica recomendada para especificaciones de requisitos de software IEEE 830.

1.2 Proposito del sistema

El presente documento tiene como propósito definir las especificaciones funcionales, no funcionales, así como las tecnologías para el desarollo de un sistema de información web que permitirá las ventas y gestión de una tienda virtual. Este será utilizado por clientes de todo el territorio nacional, asi como tambien por administradores del mismo.

1.3 Ámbito del sistema

El nombre de este sistema es TecnologyShop, y nace precisamente de la relacion entre compras y productos tecnológicos.

¿Qué hará el sistema?
Este sistema basicamente es una tienda que funcionará en un sitio web, le permitirá a los clientes poder observar diferentes productos, comprarlos, calificarlos y tambien generar una factura electronica, asi como tambien un seguimiento del producto.

Objetivos
LLegar a todas esas personas que han convertido las tiendas virtuales en sus favoritas para comprar por internet. Este proyecto no solo beneficia a los consumidores, sino también, al negocio. Por ejemplo, si el cliente considera que ahorra tiempo y dinero buscando sus productos facilmente aqui y compra en linea en cualquier momento del dia, no dudurá en continuar haciendolo. Por lo tanto, se esperá que la empresa genere grandes ingresos.

Herramientas tecnológicas
Este sistema estará basado en un sitio web, desarrollado en colaboracion con lenguajes de hipermarcado de textos como HTML, para todo lo que tiene que ver con la estructuracion del sitio; CSS para los estilos del sitio, JAVASCRIPT para las funcionalidades del sitio y MYSQL para la gestión de base de datos.
Además este sitio web estará alojado en una plataforma llamada GITHUB, propiedad de microsoft, la cual nos permite crear repositorios de código y guardarlos en la nube de forma segura, usando un sistema de control de versiones llamado Git.

1.4 Definiciones y acrónimos

Administrador: persona a cargo del proyecto, tendrá el control total sobre el mismo, pero tambien la responsabilidad de que se termine de manera exitosa.

Requerimientos: son los que definen las funciones que el sistema será capaz de realizar, describen las transformaciones que el sistema realiza sobre las entradas para producir salidas

Participantes del proyecto: se refiere a todas las personas que están involucradas directamente en la solucion del sistema.

ERS: especificacion de requisitos del sistema

Version: es el proceso de asignación de un nombre, código o número único, a un software para indicar su nivel de desarrollo.

IEEE 830: se conoce como el documento de especificación de requerimientos de software y comprende un listado de los requerimientos y del contexto de la solución, así como una descripción general del diseño por medio de los casos de uso y los escenarios.

Hardware: es toda la parte fisica que hace que un sistema pueda funcionar, ejemplo, una torre, una memoria, una tarjeta grafica.

Software: conjunto de programas y rutinas que permiten a la computadora realizar determinadas tareas.

Clientes: son aquellas personas para las cuales se ha desarrollado esta solucion, aquellas que podrán hacer uso del sitio para realizar compras.

Registro de trazabilidad: los modelos de trazabilidad facilitan que los desarrolladores realicen una correcta evaluación del costo e impacto de las posibles modificaciones en los modelos y código del software, ayudando a enlazar o relacionar los requisitos con otros elementos del ciclo de vida, principalmente, casos de prueba y código.

3.4 Restricciones

1. Uso de Dominio (X)
2. Lenguajes y tecnologías en uso: HTML, JAVA.
3. Los servidores deben ser capaces de atender consultas concurrentemente.
4. El sistema se diseñará según un modelo cliente/servidor.
5- El sistema deberá tener un diseño e implementación sencilla, independiente de la plataforma o del lenguaje de programación.
6. Interfaz para ser usada con internet.

3.5 Suposiciones y dependencias

1. Se asume que los requisitos aquí descritos son estables
2. Los equipos en los que se vaya a ejecutar el sistema deben cumplir los requisitos antes indicados para garantizar una ejecución correcta de la misma
